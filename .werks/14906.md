# Fix spurious crash reports when removing a remote site

When a remote site with enabled configuration replication is being removed from
a distributed setup, there is a brief window where the site entry exists in the
configuration with replication enabled but without a login secret. During this
window, spurious `MKGeneralException` ("The site is not logged in") crash reports
might have been recorded.
