# Fix misleading error message when accessing crash reports without permission

key        | value
---------- | ---
date       | 2026-03-10T12:19:31+00:00
version    | 2.3.0p45
class      | fix
edition    | cre
component  | wato
level      | 1
compatible | yes


Previously, when a user without the required permission tried to view or
download a crash report, a misleading "Internal error" message was shown.

The error message now correctly identifies the issue as a permissions
problem and tells the user which permission they need: 'See crash reports'.
