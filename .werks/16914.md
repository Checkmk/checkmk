# 'Changes' dialog: user setting regarding slideout and full page not being respected

version    | 2.5.0b3

edition    | community
component  | wato
level      | 1
compatible | yes

In the user profile you can set your desired behavior regarding a quick activation slideout and a full page for activating changes. If not chosen already, pressing 'Changes' will bring up the quick activation slideout and show a dialog where you can choose to just show the slideout (without being asked) or go to the full page by default instead.
However the user choice would only be stored in a browser variable and vanish after a page reload or browser restart.
This change makes the choice made in the dialog be stored in the user profile and applied correctly.
