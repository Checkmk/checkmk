# Parsing of section inv_lldp_cache failed

Previously, the inventory plugin parsing the `inv_lldp_cache` section could
crash with an `IndexError (list index out of range)`.

This happened when an LLDP neighbor advertised a port ID of sub-type
"network address" (5) together with an empty port ID value. While rendering
the address, the parser inspected the first byte to determine the address
family and failed on the empty value.

As a result, affected hosts produced a crash report and no LLDP cache
inventory data was created. The empty network address is now rendered as an
empty string and the inventory completes normally.
