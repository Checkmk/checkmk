# mongodb_locks: only discover mongodb_locks if data is available

key        | value
---------- | ---
date       | 2026-04-01T08:30:53+00:00
version    | 2.3.0p47
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

If the mk_mongodb agent plugin connects directly to a mongos instance, no
locks information is returned, which caused the _mongodb_locks_ service to go
`UNKNOWN`. This has been fixed — the service is now only discovered when locks
data is actually available.
