# Microsoft SQL Server check plugin supports empty job names

key        | value
---------- | ---

version    | 2.5.0b1
class      | feature
edition    | cre
component  | checks
level      | 1
compatible | no

In some cases, the Microsoft SQL Server agent plugin may send 
an empty job name, usually due to difficulties determining the 
codepage or because of wrong data in the job system table. 
Previously, this led to a crash; now an empty job name is 
replaced with the job ID.

You may need to rediscover services to see the new job names, but no other user action is required.
