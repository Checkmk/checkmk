# FIX ODBC backend in Microsoft SQL Server plugin reports correct job name

key        | value
---------- | ---
date       | 2025-11-26T15:11:03+00:00
version    | 2.4.0p17
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

When using the ODBC backend, the agent plugin was unable to properly 
decode non-ANSI characters in the job name. As a result, an empty name 
was sent, which made job monitoring impossible.
With this release, the agent will decode the incoming stream using 
the fallback code page Windows-1252.
