# Agent updater: do not require signature keys on remote sites in an UltimateMT
setup

key        | value
---------- | ---
date       | 2026-05-04T12:38:47.528454+00:00
version    | 2.6.0b1
class      | fix
edition    | ultimatemt
component  | wato
level      | 1
compatible | yes

Previously, on remote sites in an UltimateMT setup, the _Agent updater
(Linux, Windows, Solaris)_ ruleset rejected configurations that did not
contain at least one entry under _Signature keys the agent will accept_.
This blocked saving the rule on remote sites, where signature keys are
managed centrally.

This restores the Checkmk 2.4 behavior: the "at least one signature key"
requirement is now skipped on remote sites in an UltimateMT setup.
