# Fix negation in REST API Livestatus queries

key        | value
---------- | ---
date       | 2026-03-25T15:38:00.586276+00:00
version    | 2.5.0b3
class      | fix
edition    | community
component  | rest-api
level      | 1
compatible | yes

Negated Livestatus queries built via the `cmk.livestatus_client` Python module generated an invalid
`Negate: 1` header instead of the correct `Negate:` header (without an argument), which could lead
to unexpected query results.

REST API endpoints that accept user-defined Livestatus queries were also affected, as they use this
client internally.
