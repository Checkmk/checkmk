# Dashboards: Fix migration of view widgets with missing optional fields

key        | value
---------- | ---
date       | 2026-04-02T14:30:53.532145+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | multisite
level      | 1
compatible | yes

Previously, migrating old view widgets to the new embedded view format could fail with a
`KeyError` if the stored dashboard configuration was missing optional fields such as
`group_painters`, `browser_reload`, `num_columns`, `column_headers`, or `sorters`.

These fields now fall back to their default values during migration, so older configurations
are handled gracefully.
