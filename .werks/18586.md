# Fix DRBD agent to collect all volumes, not just volume 0

key        | value
---------- | ---
date       | 2026-04-09T14:31:09.791939+00:00
version    | 2.5.0b5
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Previously, the DRBD agent section only collected data for volume 0 from the
sysfs path, missing any additional volumes. On DRBD 9.x setups with multiple
volumes, this meant incomplete monitoring data.

The agent now collects data for all volumes.
