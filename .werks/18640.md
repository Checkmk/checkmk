# Prevention of passwords based on a local wordlist

Checkmk now prevents users from setting passwords that appear in a static common password list provided by Checkmk. When a user attempts to use a password from this list, they will receive an error and will need to choose a different, more secure password. This enhancement improves overall security by blocking weak and commonly used passwords.

This password restriction is enabled by default for all sites.

You can modify or disable this behavior in the site’s password policy under *Setup General Global > settings > Edit global setting > Prevent passwords within wordlist*.
