# Fixed Critical State for Scheduled Backups

key        | value
---------- | ---
date       | 2025-11-26T07:51:42+00:00
version    | 2.4.0p17
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

When creating a backup with scheduled execution, the related service would report a CRIT state after the backup had run. 

This is now fixed and the CRIT state will only be reported if the backup did not run at the expected time.

No manual intervention is required.
