# Add an analyze config check for automation users with stored passwords

With #17344 it was made optional to store the secret of an automation user.

With this Werk we add an *Analyze configuration* check to inform if automation users with stored secrets and critical/dangerous permissions exist.
We recommend to either not store the secret or remove the critical permissions.
