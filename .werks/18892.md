# omd: The site 'piggyback-hub' does not exist

key        | value
---------- | ---
date       | 2026-03-04T07:48:59.314410+00:00
version    | 2.4.0p25
class      | fix
edition    | cre
component  | omd
level      | 1
compatible | yes


The following `omd` commands were affected when executed with root privileges: `disable`, `mv`, `cp`, `umount`, `restore`, and `backup`.

Previously, only the `crontab` service and the `rrdcached` service determined whether the site was considered stopped.
Moreover, the following error was sometimes visible:

```
Stopping crontab...OK
Stopping xinetd...OK
Stopping stunnel...waiting for termination...OK
omd: The site 'piggyback-hub' does not exist. You need to execute omd as root or site user.
```

With this change, `omd` will now always correctly determine whether a site is stopped.
