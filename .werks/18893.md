# omd: OPENSSL not found when trying to reload Apache

---------- | ---
date       | 2026-03-04T08:16:52.786270+00:00
version    | 2.5.0b3
class      | fix
edition    | community

compatible | yes

Previously, `omd` could fail to reload the system Apache.
This issue occurred in case of a mismatch between the OpenSSL version included in the Checkmk distribution and the OpenSSL version required by the system Apache.
As a result, the error `/usr/bin/systemctl: /omd/sites/v240/lib/libcrypto.so.3: version OPENSSL_3.4.0 not found` would appear in the output of `omd`.
