# omd restore: Restoring from stdin raises exception

key        | value
---------- | ---
date       | 2026-04-01T09:34:45+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | omd
level      | 1
compatible | yes

This fixes an issue only affecting users of the Checkmk 2.5 beta.

Previously, `omd restore` raised an exception when restoring from stdin.
In particuler, restores on the on the Checkmk Appliance failed.
The following error would be displayed.

```
$ cat omd.backup.tar | omd restore -
Error while sending data to restore process: [Errno 32] Broken pipe
...
OSError: [Errno 29] Illegal seek
Site restore failed
```

This has been fixed.
