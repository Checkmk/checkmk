# Fix installation failure on AlmaLinux 9 due to GLIBC_2.35 symbol mismatch

Previously, Checkmk packages for AlmaLinux 9 could fail to install on systems pinned
to a minor version below 9.7 with an error like:

    ImportError: /lib64/libc.so.6: version `GLIBC_2.35' not found (required by
    /omd/versions/2.4.0p22.cee/lib/python3.12/lib-dynload/.../.so.1)

AlmaLinux 9.7 backported glibc 2.35 symbols into its glibc 2.34 binary. When the
Checkmk build environment resolved its live package mirror to 9.7, shared libraries
compiled during the build would require `GLIBC_2.35` symbols — which do not exist on
AlmaLinux 9.6 and earlier.

The build environment is now pinned to the vault.almalinux.org/9.2 snapshot, ensuring
Checkmk packages are always compiled against a baseline glibc 2.34 without the
backported 2.35 symbols. Packages will now install correctly on all supported
AlmaLinux 9 minor versions.
