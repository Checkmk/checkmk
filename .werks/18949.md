Fixed monitoring of remote piggyback hosts

With this werk we resolved an issue where piggyback hosts located on one remote site
could not be monitored on a different remote site, even when both sites shared the same customer.

This was due to an inconsistency in how configurations were synced across the distributed setup.

A configuration sync must be performed across all affected sites.
To facilitate this, the user should introduce and activate a change; a simulated change
(editing and saving a host without modifications) is sufficient to trigger the synchronization process.
