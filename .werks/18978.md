# Add monitoring support for APC NetShelter Advanced Rack PDUs

Checkmk adds monitoring for APC NetShelter Advanced Rack PDUs (APDU series).
All plugins auto-detect devices via SNMP where sysObjectID starts with
`.1.3.6.1.4.1.318.1.1.32`.

The `apc_netshelterpdu_power` plugin monitors electrical current and power
consumption at device, phase, and bank levels. The device service also reports
total load as a percentage of rated capacity (default thresholds: WARN at 80%,
CRIT at 90%). Device-reported current thresholds are applied automatically per
phase. The _Electrical Phase_ ruleset can be used to override thresholds.
A new optional _Load_ parameter has been added to the _Electrical Phase_
ruleset to allow configuring total load thresholds.

The `apc_netshelterpdu_outlet` plugin reports individual outlet on/off status
as `OK` when on and `WARN` when off.

The `apc_netshelterpdu_inventory` plugin collects hardware and software
inventory data including device model, serial number, firmware version,
manufacture date, and electrical rating.

To start monitoring these devices, run a service discovery on the affected
hosts after updating.
