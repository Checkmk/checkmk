# Fix time zone handling for autodiscovery excluded time windows

Checkmk allows configuring time windows during which automatic host
discovery should not run. Due to a bug, the Checkmk server's local time
zone was not taken into account when evaluating these windows: the
comparison was done against UTC rather than local time. On servers not
running in UTC, this caused the exclusion windows to be applied at the
wrong hours.

This has been fixed by using local time for the comparison.

If your Checkmk server is not running in UTC and you have configured
excluded time windows for automatic discovery, you should review those
windows to ensure they reflect the intended local times.
