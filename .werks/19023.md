# Added Seconds to Timerange in Elasticsearch Query Rule

key        | value
---------- | ---
date       | 2025-11-26T09:28:27+00:00
version    | 2.4.0p17
class      | feature
edition    | cre

level      | 1

The active check now supports specifying intervals in seconds instead of minutes, allowing users to configure more precise timing. Existing rules that use minutes will continue to function as before, and no manual intervention is required unless users want to take advantage of the new second-level granularity.
