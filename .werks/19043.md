# Crash when displaying rules with wildcard host name conditions

key        | value
---------- | ---
date       | 2026-03-16T13:54:42.622408+00:00
version    | 2.3.0p47
class      | fix
edition    | cre
component  | wato
level      | 1
compatible | yes

The rule listing page in Setup crashed with a
`ValueError: Invalid hostaddress` error when displaying rules that contained
explicit host name conditions using wildcard characters (e.g. AP-SEDE-TCOTILLAS*).

This issue has been fixed. Such host specifications are now rendered as plain text,
preventing the crash.
