# Fix Dell Compellent disk health state: not healthy reported as UNKNOWN

key        | value
---------- | ---
date       | 2026-03-24T11:58:53.829562+00:00
version    | 2.5.0b4
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Previously, the _Dell Compellent Disks_ check incorrectly reported disks in
a not-healthy state as `UNKNOWN` instead of `CRIT`. The SNMP health value `2`
(not healthy) was not recognized by the health state mapping — it expected `0`
— causing the check to fall back to the unknown state.

This has been fixed. Disks reporting SNMP health value `2` are now correctly
flagged as `CRIT`.
