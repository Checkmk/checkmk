# Make piggyback datasource always transparent

In some cases, agents running on a host can produce piggyback data for exactly this host. 
In the site, this fact was not made transparent;
instead this piggyback data was treated like regular agent data.

With this change, piggyback data produced on a host for itself is treated as such.
As a consequence, the `cmk/piggyback_source_<hostname>:yes` label is added to that host accordingly.
