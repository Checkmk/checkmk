# Servicenow: Fix missing error details on failures

key        | value
---------- | ---
date       | 2026-03-10T08:25:07.976868+00:00
version    | 2.5.0b1
class      | fix
edition    | pro
component  | notifications
level      | 1
compatible | yes

Previously, when the ServiceNow notification plugin failed with exit code 2,
no error details were shown to the user, making it impossible to diagnose the
root cause.

Error details from failed ServiceNow API requests are now properly written to
stdout so they appear in the notification log.
