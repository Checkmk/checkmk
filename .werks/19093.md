# Sub reports: Inherit time range from parent report

key        | value
---------- | ---
date       | 2026-03-13T08:58:40.413117+00:00
version    | 2.4.0p25
class      | fix
edition    | cee
component  | reporting
level      | 1
compatible | no

Embedded child reports now inherit the time range from their parent reports.

Previously, when a child report was embedded in a parent report, it ignored the
parent's time range and instead used its own configured time range or the
default of the last 24 hours. This change ensures that embedded child reports
display data consistent with the parent report’s time frame, preventing
potential confusion caused by mismatched time ranges.

Users should verify that their existing reports with embedded sub reports
generate the expected time ranges.
