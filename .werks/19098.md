# Fix empty error message when scheduled report email delivery fails

key        | value
---------- | ---
date       | 2026-03-17T05:47:35.552369+00:00
version    | 2.4.0p26
class      | fix
edition    | cee
component  | reporting
level      | 1
compatible | yes

Previously, when sending a scheduled report via email failed (e.g. due to
mailbox size limits or other mail delivery errors), the error message was
empty.

The actual error from the mail transfer agent was lost because the exception
was re-raised without passing the original message. You now get the actual
reason for the failure both in the log and in the schedule's error display.
