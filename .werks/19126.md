# Fix OTel Sum metric rate computation for delta temporality

key        | value
---------- | ---
date       | 2026-03-27T06:19:02.506659+00:00
version    | 2.4.0p26
class      | fix
edition    | cce
component  | metrics
level      | 1
compatible | no

Previously, when rate computation was enabled for OTel Sum metrics (e.g., with
"always" or "monotonic counter" mode), Checkmk assumed cumulative temporality
for all Sum metrics. This was incorrect for delta-temporality metrics, where
the reported value already represents the change over the reporting period
(the interval between start and end timestamp).

Checkmk now checks the temporality of OTel Sum metrics at check time. For
delta-temporality metrics, the rate is computed directly from the value and
its reporting period, without requiring historical state. For
cumulative-temporality metrics, behavior is unchanged.

Because of this change, existing thresholds based on delta values may no longer trigger alerts correctly.
If you use OpenTelemetry metrics with "sum" metrics in your Checkmk collector, review and update all thresholds configured in the **OpenTelemetry** service monitoring rule to reflect the new values.
