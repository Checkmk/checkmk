# OTel: RED signal checks add 24-hour aggregation window

key        | value
---------- | ---
date       | 2026-03-31T15:13:52.846312+00:00
version    | 2.5.0b3
class      | feature
edition    | ultimate
component  | checks
level      | 1
compatible | yes

The RED signal checks (Errors, Latency, Requests) for OpenTelemetry-monitored services now include an additional "Last day" (24-hour) aggregation window, alongside the existing "Last minute" and "Last hour" windows.

This provides a longer-term view of service health trends for the following checks:

- Errors
- Latency
- Requests

No configuration changes are required. The new 24-hour signal appears automatically after updating.

These checks were originally introduced in [werk #18733](https://checkmk.com/werk/18733).
