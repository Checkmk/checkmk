# Fix crash when editing backup jobs with hypenated target IDs

class      | fix

Previously, editing a backup job whose target had a hyphenated identifier (e.g. `backup-local`)
caused a crash with the error:

_ValueError: 'backup-local' is not a valid, non-reserved Python identifier_

This affected backup targets created in Checkmk 2.3 or 2.4 that used hyphens in their target ID.
Starting and completing backups still worked — only the "Edit this backup job" page was broken.
