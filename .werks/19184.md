# Only parse required keys for Bazel plugins

key        | value

version    | 2.4.0p25
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

Crashes were reported in the Bazel plugin family when the section parser failed
to parse a number.

These crashes occurred only in fields unrelated to those required by the
service.

To address this, the Bazel plugin family now specifies which keys should be
parsed as numbers, while ignoring all other keys. This fix reduces crashes
caused by parsing errors in non-essential fields, improving monitoring
stability. Users do not need to take any action.
