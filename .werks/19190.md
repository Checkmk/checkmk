# Prevent crash of Windows agent if WMI is not accessible

key        | value
---------- | ---
date       | 2026-03-02T14:01:01.950006+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | checks
level      | 2
compatible | yes

Previously, the Windows agent would crash and restart when WMI
(Windows Management Instrumentation) was unavailable — a rare but
possible situation on some Windows systems.

This has been fixed. The agent now handles WMI connectivity failures
gracefully. No action is required.
