# rrd: Skip RRD creation for filenames exceeding filesystem limits

key        | value
---------- | ---
date       | 2026-04-17T10:39:35+00:00
version    | 2.5.0
class      | fix
edition    | community
component  | core
level      | 1
compatible | yes

Nagios core now skips RRD file creation when the generated filename would exceed
the filesystem `PC_NAME_MAX` limit. This can happen with services that have long
descriptions or metric names.

Previously, attempting to create such files could cause silent failures or crashes
in the GUI. The affected service is now skipped with a log warning instead, keeping
the monitoring core stable.

No action from the user is required.
