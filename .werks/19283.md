# corosync latency: prevent links from disappearing when no latency samples are present

key        | value
---------- | ---
date       | 2026-03-24T14:28:24.605243+00:00
version    | 2.5.0b2
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Discovery now always includes connected corosync latency links, even when links had no associated latency samples. Previously, discovery skipped these links if they were still marked as connected, which in some setups caused corosync latency services to disappear occasionally.

With this change, users will no longer experience unexpected disappearance of corosync latency services, resulting in more stable and reliable monitoring. No user action is required to benefit from this improvement.
