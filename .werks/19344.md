# Metric backend: cap maximum memory limit at 90%

key        | value
---------- | ---
date       | 2026-05-05T07:41:53.151893+00:00
version    | 2.6.0b1
class      | feature
edition    | ultimate
component  | wato
level      | 1
compatible | yes

The metric backend's _Memory limit_ setting can no longer be configured above
90%. The previous upper bound of 200% allowed configurations that could starve
the rest of the monitoring stack of memory on the same host.

If you currently have this value set above 90% please edit the rule and
set the value within the supported range.
