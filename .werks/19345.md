# Custom graph editor: Fix z-fighting between table header and rows

key        | value
---------- | ---
date       | 2026-05-06T06:30:28.690229+00:00
version    | 2.5.0p2
class      | fix
edition    | community
component  | multisite
level      | 1
compatible | yes

Previously, when editing a custom graph with multiple lines, the sticky table
header at the top of the graph lines table appeared semi-transparent while
scrolling causing graph rows to show through the header.

This has been fixed: the header is now opaque and stays on top of the rows,
while per-row dropdown popups (e.g. for _Line style_) still open above the
header as expected.
