# Fix metric translations for check_ping.exe and check_tcp.exe

Werk #16254 added metric translations for the Windows-style Nagios plug-ins
`check_ping.exe` and `check_tcp.exe`, but the registration key was not
normalized the same way as the lookup key. As a result, the translations
never fired:

* The `rta` metric reported by `check_ping.exe` was rendered in seconds
  instead of milliseconds (values appeared ~1000× too large).
* The `time` metric reported by `check_tcp.exe` was not renamed to
  `response_time`.

Both translations now apply correctly.
