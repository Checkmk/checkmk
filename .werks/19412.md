# OTel: Monitor data flow of collector

The `OMD SITE_NAME OTel collector` service previously only allowed tracking of memory usage, as announced in [#werk 18528](https://checkmk.com/werk/18528),

To help debug issues when data is missing or not displayed correctly, the service now also provides information about the metrics and logs received.

Alerting based on this additional information is not yet supported.
