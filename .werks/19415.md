# Fix piggybacked host not handled correctly when source identifies it by IP address

---------- | ---
date       | 2026-03-19T17:42:34.847471+00:00
version    | 2.5.0b3
class      | fix
edition    | community
component  | core
level      | 1
compatible | yes

Previously, if a piggyback source identified a piggybacked host by its IP address
instead of its hostname, it was handled incorrectly in Checkmk:

- The host was not identified as a piggybacked host, meaning the relevant services would not be discovered.
- Downloading the agent output yielded empty data for such hosts

All affected lookups now also check the configured IPv4 and IPv6 addresses of
the host, so piggyback data sent under either the hostname or any configured
IP address is found correctly.
