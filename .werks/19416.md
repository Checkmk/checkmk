# f5_bigip_pool: Fix services showing as changed after update

key        | value
---------- | ---
date       | 2026-04-02T09:27:43.181265+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Previously, when upgrading Checkmk, _Load Balancing Pool_ services monitored
by the F5 BIG-IP pool check could appear as changed services in the discovery
view after the update, showing removed discovery parameters.

This is due to the check plugin now using built-in defaults for parameters
instead of discovering them, without the stored parameters being updated as well.

The stored discovery parameters are now automatically dropped during the update
to avoid changing during discovery later on.
