# Agent Bakery: Allow configuring Windows agent communication channel

key        | value
---------- | ---
date       | 2026-03-05T15:56:31.946803+00:00
version    | 2.4.0p24
class      | feature
edition    | cee
component  | agents
level      | 1
compatible | yes

You can now configure the internal communication channel between the Windows
agent and the Agent Controller directly through the Agent Bakery.

A new option _Internal agent communication channel_ is available in the
_Windows Agent Controller_ ruleset. By default, a mailslot is used for local
communication. If you encounter problems with mailslot communication in your
environment (e.g., due to security policies), you can switch to a TCP socket
on localhost instead, with a configurable port (default: 28250).

Previously, the only way to change this was to manually edit
`check_mk.user.yml` on every monitored host.

**Note:** We recommend leaving this on the default (Mailslot) unless you
actually encounter problems.
