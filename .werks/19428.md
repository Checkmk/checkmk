Previously, the active checks _Check email delivery_ (`check_mail_loop`), _Check Email_ (`check_mail`), and _Check mailboxes_ (`check_mailboxes`) crashed with an `AssertionError` when using POP3 as the fetch protocol:

_AssertionError: Expected tuple with two elements, got (b'+OK 2 messages:', [...], 17)_

This is a regression introduced in Checkmk 2.4. IMAP connections were not affected.

## Technical background

An internal refactoring of the email response parser introduced assumptions about the response format that were incompatible with Python's `poplib` library. This has been fixed.
