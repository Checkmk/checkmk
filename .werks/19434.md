# Hitachi HNAS FC interfaces: handle missing error counter columns

Hitachi HNAS hosts could end up with no _Interface FC_ services, accompanied by _Parsing of section hitachi_hnas_fc_if failed - please submit a crash report!_ in `WARN` state. The check plugin's parser crashed when the underlying SNMP data only contained an empty string for any of the Fibre Channel error or discard counter columns.

The crash was observed on a Hitachi VSP One Block running firmware 15.5.x. That device matches the HNAS SNMP detect via the BlueArc enterprise OID but only partially implements the BlueArc-Titan fcStatistics table, leaving several error and discard columns empty. Any Hitachi-family device that does not populate every error counter column was affected the same way.

Missing counters are now reported as unknown rather than forced to zero. The per-port `if_in_errors` and `if_in_discards` metrics — visible in the GUI as the _Input errors_ and _Input discards_ lines of the _Errors (discards)_ graph — are simply omitted for affected ports rather than reading as silent zeros that could mask real CRC or discard storms. Classic HNAS devices populate every column and are unaffected by the change.
