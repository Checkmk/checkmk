# Synthetic Monitoring: Root agent deployment on Linux is no longer supported in combination with RCC as environment creation mode

key        | value
---------- | ---
date       | 2026-05-03T12:27:05.743140+00:00
version    | 2.4.0p33
class      | fix
edition    | cee
component  | checks
level      | 1
compatible | no

When RCC (used by Synthetic Monitoring) runs as root on Linux and the RCC home
directory (e.g. `/opt/robocorp`) does not yet exist, RCC may change the
permissions of `/opt` from `555` (the default on SLES 16) to `750`. This breaks all workflows where
non-root users need to traverse `/opt`, including Checkmk sites installed under
`/opt/omd` — as observed on SLES 16.

Baking a Linux agent package that combines root agent deployment with at least
one RCC-based plan now fails with a clear error message instead of producing a
broken package.

**Migration:** Switch the affected hosts to non-root agent deployment
via the rule _Customize agent package (Linux)_.
