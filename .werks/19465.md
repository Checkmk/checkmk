Previously, replacing a managed robot's file would fail with an error like
"Robotmk scheduler (Linux) — Please provide an absolute Windows path."
This happened whenever at least one _Robotmk scheduler (Windows)_ bakery
rule existed in the system — even if it didn't reference the robot being
replaced.

This has been fixed.
