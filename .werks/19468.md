# Fix blank subscription entries and validate IDs in Azure v2 Quick Setup

version    | 2.5.0b3
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

Previously, the Azure v2 Quick Setup allowed you to save blank entries in the
_Explicit list of subscription IDs_ field and blank values in the
_Tag matching subscriptions → is_ condition. Both now reject empty input with
a validation error.

Additionally, the Step 1 connection test now verifies that each explicitly
specified subscription ID actually exists in your Azure tenant. If a
subscription is not found, the test fails immediately with a clear error
message, rather than silently succeeding and failing later during monitoring.
