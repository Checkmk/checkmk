# Azure App Registration: ignore credentials older than N days

Azure App Registration secrets and certificates can now be ignored if
they expired more than a configurable threshold ago.

This is useful when an expired credential has already been replaced but
still appears in Azure — instead of getting a CRIT alert for it, you can
suppress it by setting an age limit.

Configure the threshold under **Setup > Services > Service monitoring
rules > Azure App Registration**, in the *Check secret credentials* or
*Check certificate credentials* section, via the new **Ignore if older
than** option. A credential that expired more than the threshold ago is
reported as OK with a brief note.
