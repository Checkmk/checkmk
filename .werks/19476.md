# Fix missing Azure agent info service on Azure v2 tenant host

class      | fix

level      | 1
compatible | no

Previously, the _Azure agent info_ service was not always created on the Azure v2 tenant host. The special agent only wrote the `azure_v2_agent_info` section when an error or exception occurred during data collection. If everything ran cleanly — most noticeably when monitoring only app registrations — the service was not discovered, and the health check was simply absent.

The agent now always emits an `azure_v2_agent_info` section, so the service is reliably present on the tenant host regardless of the monitoring configuration.

Users must run a discovery to start monitoring the _Azure agent info_ service.
