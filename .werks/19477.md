# LDAP: User Synchronize for Active Directory can unlock users

key        | value
---------- | ---
date       | 2026-03-17T14:34:49+00:00
version    | 2.4.0p26
class      | fix
edition    | cre
component  | wato
level      | 1
compatible | yes

User synchronization with an LDAP connection using Active Directory manages user login status based
on the AD account settings. Previously, if a user’s login had been disabled in Checkmk because their
AD account was marked as "Account is disabled," this status would not be updated during subsequent
synchronizations.

As a result, even after the user’s AD account was enabled again, their login in Checkmk remained
disabled.

As of this werk, user synchronization with Active Directory will correctly update for users whose
account have been reactivated in AD.
