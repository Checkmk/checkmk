# activate changes slideout: keep number of pending changes call running on site restarts

The sidebar’s number of pending changes badge was previously not updated after an activation requiring a site restart. This could cause confusion, as the badge might display an incorrect number of pending changes, making it unclear whether all changes had been fully applied.

This werk ensures that the number of pending changes shown in the sidebar is accurately updated after such activations. Users can now trust that the badge reflects the current state without needing any manual refresh or action.
