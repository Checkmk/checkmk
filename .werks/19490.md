# Activate changes slideout: increase the timeout and update pending changes on activation failure

key        | value
---------- | ---
date       | 2026-03-19T11:11:28.255694+00:00
version    | 2.5.0b2
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

Previously, the activate changes mechanism could incorrectly indicate that changes had not yet been applied, even though the activation had already completed successfully. This was caused by insufficient waiting time
and unreliable status checks during the activation process.

We increased the timeout and improved the status verification logic to
ensure that the activation state is reported accurately. No user action is required.
