# activate changes: on activate changes failures, request latest changes

When activating changes on the slideout fails due to a Rest-API error, the user will see an error message indicating the failure.

In response, the system now automatically fetches the latest pending changes again. 

This update addresses situations where another user may have already activated changes, ensuring that the slideout always displays the current pending changes. Users can then review the updated state before attempting to activate changes again.
