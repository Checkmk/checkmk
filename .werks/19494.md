# Fix audit log showing wrong role name after cloning a user role

Previously, when cloning a user role (e.g. cloning 'user' to create 'userx'),
the audit log entry on the _Activate Changes_ page incorrectly showed the
original role name instead of the newly created role's name. This has been fixed.
