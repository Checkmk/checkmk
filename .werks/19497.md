# SAML: Users with no matching role mapping now receive default roles instead of empty roles

key        | value
---------- | ---
date       | 2026-03-18T14:18:12.979297+00:00
version    | 2.3.0p46
class      | fix
edition    | cee
component  | setup
level      | 1
compatible | yes

When SAML role membership mapping was configured but the IDP-provided roles did
not match any mapping, users ended up with an empty roles list. Now, if no roles
can be mapped, the default user profile roles from the global settings are
assigned instead.
