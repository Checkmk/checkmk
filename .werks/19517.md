# Fix search filters not shown in filter sidebar

key        | value
---------- | ---
date       | 2026-03-24T13:19:33.194457+00:00
version    | 2.4.0p26
class      | fix
edition    | cre
component  | multisite
level      | 1
compatible | yes

Previously, when using filter expressions in "Search in monitoring" (e.g. `st:CRIT`,
`h:myhost`, `s:CPU`) and pressing Enter to open the results as a view, the applied
filters were missing from the filter sidebar. This made it unclear which filters were
active. Additionally, manually adding the corresponding filter via "Add filter" showed
default values instead of the currently applied ones.

The search now correctly passes the active filter information to the view, so the
filter sidebar properly reflects all applied filters with their current values.
