# Fix crash when saving a configuration bundle that no longer exists

Previously, saving a configuration bundle (via the _Parameters for service_
page) could crash with an AttributeError if the bundle had been deleted
between the time the form was loaded and the time the save was submitted —
for example, due to a concurrent deletion or a browser tab left open.

The page now handles this case gracefully and redirects back instead of
crashing with an internal error.
