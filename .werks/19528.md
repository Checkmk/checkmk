# Fix HyperV Checkpoints crash on negative checkpoint age

key        | value
---------- | ---
date       | 2026-03-25T07:49:20.064628+00:00
version    | 2.5.0b2
class      | fix
edition    | community

Previously, the _HyperV Checkpoints_ service crashed with a
`ValueError: Cannot render negative timespan` when the HyperV agent
reported a negative checkpoint age. This can happen due to clock skew
between the monitored host and the Checkmk server.

The service now reports `WARN` with a message indicating the negative
age and advising to check for clock skew.
