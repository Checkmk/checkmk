# Host check: fall back to the Check_MK service status when smart ping is not possible

For hosts monitored by the Checkmk Micro Core, the default host check
is _smart ping_. When smart ping is not possible — for example, because
the host has no resolvable IP address (typical for hosts that push their
data or are registered via agent registration) — Checkmk previously fell
back to a regular ICMP ping, which would not succeed either.

The host check now derives the host state from the built-in Check_MK
service — equivalent to selecting _Use the status of the Check_MK Service_
in the _Host check command_ ruleset.

## How to restore the previous behavior for non-relay-monitored hosts

Configure a rule in _Setup → Hosts → Host monitoring rules → Host check
command_ for the affected hosts and pick _PING (ICMP echo request)_ (or
any other explicit choice). Explicit rules are honored; the new fallback
only applies when no rule is configured and smart ping is not possible.

Note: relay-monitored hosts always derive the host state from the
Check_MK service; explicit _PING (ICMP echo request)_ or _Smart ping_ rules are
overridden with a warning, and no rule in the _Host check command_
ruleset can restore ping behavior on such hosts.
