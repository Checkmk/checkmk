# Monitored on relay: "No relay" override was ignored for hosts in a relay folder

key        | value
---------- | ---
date       | 2026-06-18T17:50:17.677501+00:00
version    | 3.0.0b1
class      | fix
edition    | ultimate
component  | relay
level      | 1
compatible | yes

When a folder was configured to monitor its hosts via a Checkmk Relay, a host
or subfolder that overrode the *Monitored on relay* attribute to *No relay* was
still treated as a relay host. Running a service discovery on such a host failed
with:

```
[agent]: This data source is not supported for relay hosts.
```

The override is now respected: a host or subfolder explicitly set to *No relay*
is no longer monitored via the relay, and its agent and other data sources work
as expected. This also applies when the *No relay* setting is inherited from a
subfolder that overrides a relay set on a parent folder.
