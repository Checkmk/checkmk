# Fix crash in GCP Quick Setup connection test

When setting up Google Cloud Platform monitoring via the Quick Setup,
testing the connection failed with a crash instead of validating your
service account credentials:

_the JSON object must be str, bytes or bytearray, not NoneType_

This affected all connection test attempts where the credentials are
stored in the password store, which is the case for any setup done
through the user interface. The actual data collection was not affected;
only the connection test in the Quick Setup crashed.

This has been fixed. The connection test now reads the service account
credentials from the password store, just like the regular agent run
already did.
