# Fix notification and alert timeline widgets ignoring time range setting

version    | 2.5.0b3
class      | fix

compatible | yes

Previously, the "Problem notifications" and "Problem alerts" dashboard widgets
could display an incorrect time range on the x-axis. Instead of showing only the
configured time window (e.g. 25 hours), the chart could stretch to cover a much
wider period. This was especially noticeable when the server time and browser time
differed.

The widgets now correctly respect their configured time range setting.
