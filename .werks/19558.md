# Widget settings lost when re-editing a dashboard widget

When editing an existing dashboard widget, some settings were reset to their
defaults instead of showing the previously saved values:

- The "Show widget background" checkbox always appeared unchecked, even if it
  had been enabled before.
- The "Link content to" target showed "not selected" instead of the previously
  configured dashboard or view.

Both settings now correctly restore their saved values when re-opening the
widget configuration.
