# Room Alert 3S: fix internal temperature sensor when Celsius unavailable via SNMP walk

---------- | ---
date       | 2026-04-28T12:28:03.482135+00:00
version    | 2.5.0p6
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

The Room Alert 3S internal temperature sensor plugin (introduced in Werk #19562)
only fetched the Celsius OID from the device. On some hardware configurations,
direct SNMP walks do not return a Celsius reading, causing the
**Room Alert 3S Temperature** service (item: **Internal**) to produce no data.

The plugin now fetches both the Celsius and Fahrenheit OIDs. Celsius is preferred
when available; Fahrenheit is used as a fallback so that the service continues to
report the internal temperature correctly.
