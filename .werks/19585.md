# Fixed various edge cases of the availability calculation

The availability calculation did not handle various edge cases correctly,
most prominently the handling of nested and/or overlapping downtimes. Other
edge cases were related to core restarts and rotations of the monitoring
log. The most visible symptoms were downtime phases in the availability
timeline which ended too early or never. All of this has been fixed.
