# Fixed crash in RRD creation for service names containing a null byte

class      | fix

compatible | yes

Previously, if a service description contained an embedded null byte — for
example a stray `NUL` character emitted by some SNMP devices in a service name
like _PDU Bank B6_ — the creation of the RRD database for that service crashed
with the following error, and a crash report was generated:

_ValueError: embedded null byte_

As a result, no performance data (and therefore no graphs) could be stored for
the affected service.

This has been fixed. The service name is now sanitized before it is used to
build the RRD file path, so RRD creation no longer crashes and the performance
data for the affected service is stored again.
