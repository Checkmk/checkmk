# REST API: fix required parameter marking in OpenAPI spec

Since 2.5.0, some REST API endpoints did not always mark required path and query parameters as such in the OpenAPI specification.
This affected only a subset of endpoints. Tools consuming the spec, such as Swagger UI, API clients, or SDK generators, could therefore treat mandatory parameters as optional for those endpoints.
