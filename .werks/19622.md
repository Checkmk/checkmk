# Subreports now inheriting heading levels

When a report contained more than one "Host report for multiple hosts" (or "Service report for multiple services")
element, headings inside the second and subsequent subreport blocks were numbered incorrectly. This caused headings
to appear with wrong section numbers, including spurious .0. segments (e.g. 3.0.1.1 instead of 3.1.1).

The issue affected any report where subreport elements appeared consecutively without a heading element between
them.

This werk now fixes the problem.
