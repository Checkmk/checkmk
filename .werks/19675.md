# Cisco Meraki: handle non-integer switch port ids

key        | value
---------- | ---
date       | 2026-04-02T09:58:29.942957+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

The Meraki API began returning non-integer port IDs for some of its newer
devices. This lead to crashes during the parsing phase of the _Switch Ports
Statuses_ check and inventory plugins. After some analysis, we have decided to
skip these entries as they appear to serve only as placeholders that don't add
value to our existing plugins.

There is nothing to do from the customer side outside of updating.
