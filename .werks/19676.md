# Add grouping to single aggregation and availability views

key        | value
---------- | ---
date       | 2026-04-07T12:54:15+00:00
version    | 2.6.0b1
class      | fix
edition    | cre
component  | bi
level      | 1
compatible | yes

The URLs for the icon links to the "Single Aggregation" and "Availability of"
views were missing the aggregation group information on the "All aggregations"
page. This caused the entries to be duplicated for both of these views, leading
to a confusing user experience. This information is now a part of the respective
icon URLs, the user will be able to clearly see which entry belongs to which
group.

As this is simply a GUI fix, there is no manual intervention required by the
user.
