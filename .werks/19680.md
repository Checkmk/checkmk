# omd rm/mv: Prevent broken state when site home is mounted

key        | value
---------- | ---
date       | 2026-04-10T08:22:42.824205+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | omd
level      | 1
compatible | yes

When running `omd rm <site>` on a site with a mounted home directory, the site
would only be partially deleted: the site user account is removed but the
mounted directory is left behind. Subsequent calls to `omd rm` would then fail
because the site user no longer exists.

When running `omd mv <site> <new-site>` on a site with a mounted home
directory, the new site would only be partially created: the new user account
is created but the site directory rename fails because the directory is still
mounted. Subsequent calls to `omd mv` would then fail because the new site user
already exists.

Both commands now abort early with an error message asking the user to unmount
the site home directory first.
