# Natural sort: handle empty strings without crashing

key        | value
---------- | ---
date       | 2026-04-23T07:52:06.447711+00:00
version    | 2.5.0p1
class      | fix
edition    | community
component  | multisite
level      | 1
compatible | yes

Opening the Setup > Hosts page crashed with `IndexError: string index out of
range` if the folder contained a host whose name was empty (for example due
to a corrupted host definition). The natural-sort helper used to build the
host list accessed the first character of every key unconditionally.

Empty keys are now treated as symbols (same sort bucket as keys starting
with underscore), so the page renders instead of aborting.
