# omd: Ensure only one backup runs at a time

key        | value
---------- | ---
date       | 2026-05-22T08:11:51.988624+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | omd
level      | 1
compatible | yes

Previously, running `omd backup` while a backup was already in progress on the
same site would silently start a second concurrent backup. Concurrent backups
are now prevented. A second invocation will fail immediately with an error.

This also fixes a crash introduced on SLES 16.0 (shipped with systemd 257),
where the `omd` command would fail with:

```
FileNotFoundError: [Errno 2] No such file or directory: '/run/lock/mkbackup'
```

SLES 16.0 no longer creates `/run/lock/` on fresh installs before first boot.
The lock file used to coordinate backups has been moved to the site directory,
which is always available.
