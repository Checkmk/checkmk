# Fix stale download token in agent slideout

key        | value
---------- | ---
date       | 2026-04-10T08:13:47.343957+00:00
version    | 2.5.0b5
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

Previously, when generating a one-time token on one OS tab (e.g. Linux) in the
agent installation slideout and then switching to another tab (e.g. Windows) or
changing the package format (e.g. from Debian to RPM), the token from the
previous selection persisted. This caused the download commands to appear
immediately with a stale token. Now switching tabs or changing the package
format resets the token, so you need to generate a fresh one.
