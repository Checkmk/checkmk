# Personal notification rules: Synchronize to remote sites

key        | value
---------- | ---
date       | 2026-04-27T12:00:00+00:00
version    | 2.4.0p27
class      | fix
edition    | cre
component  | notifications
level      | 1
compatible | yes

In a distributed setup, creating, editing, deleting or moving a personal
notification rule did not synchronize the change to the remote sites.
The rule was saved on the central site, but remote sites stayed unaware
of it until another event (such as the next "Activate Changes" or a
re-login) eventually triggered the user profile replication.

As a result, notifications could behave inconsistently between sites,
and the affected rule did not appear when testing notifications on a
remote site.

After saving a personal notification rule, the synchronization to the
remote sites is now started right away as intended.
