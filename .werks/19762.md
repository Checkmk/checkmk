# User notification rules: Fix "User ID must not be None" error when saving rule

key        | value
---------- | ---
date       | 2026-04-28T12:30:24.894971+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | notifications
level      | 1
compatible | yes

Saving a user-specific notification rule from "Setup > Users > Notification
rules of user" failed with an "User ID must not be None" error message.
After saving, the action redirected back to the user notification rules
overview without forwarding the affected user ID, so the target page could
not determine which user it should display.

The user ID (and an active search query) are now preserved on the redirect
after saving a user notification rule.
