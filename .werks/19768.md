# Logwatch: clarify "Clear log" confirmation dialog

The confirmation dialog shown when acknowledging logwatch messages
("Clear log") used to read "This affects the log file &lt;file&gt; on
host &lt;host&gt;". This wording suggested that the action removes the
log file on the monitored host, while in reality only the messages
stored on the Checkmk site are acknowledged and removed.

The dialog now makes this explicit:

"Clearing the log acknowledges and removes the stored messages.
Log files on the host stay unchanged."

The confirm button label has been adjusted accordingly to "Clear log"
(or "Clear logs" when acknowledging multiple log files).
