# Fix loading skeleton shown after cancelling page navigation

key        | value
---------- | ---
date       | 2026-04-10T11:24:08.526145+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

Previously, if a loading transition (skeleton screen) was triggered and the
user then cancelled a navigation dialog ("Leave page?"), the loading skeleton
would still be injected into the page — leaving the UI in a broken state.

This has been fixed. The page now remains usable if the user decides to stay.
