# Fix unhandled OSError on client disconnect during request upload

key        | value
---------- | ---
date       | 2026-04-13T06:29:10.836989+00:00
version    | 2.5.0
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

Previously, if a client disconnected mid-upload while submitting a request to
Checkmk (e.g. aborting a large file upload), the web interface could produce
an unhandled internal error instead of returning a proper HTTP 400 response.

This has been fixed. Checkmk now correctly returns a 400 response in this case.
