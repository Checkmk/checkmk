# show_host endpoint: use contact groups

key        | value
---------- | ---
date       | 2026-04-15T11:53:48.223550+00:00
version    | 2.4.0p28
class      | fix
edition    | cre
component  | rest-api
level      | 1
compatible | yes

When trying to show information of a specific host with the
```
GET /objects/host_config/{host_name}
```
endpoint, you had to have the `See all hosts and folders` permission.

This was not aligned with the
```
GET /domain-types/host_config/collections/all
```
where all of the contact hosts have been shown.

Now the permission handling correctly uses contact groups: a user can view a
host if they are a contact of the host itself or of the folder containing it.
