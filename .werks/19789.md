# Fix response time levels losing millisecond values in Check TCP port connection rule

key        | value
---------- | ---
date       | 2026-05-18T08:46:46.777421+00:00
version    | 2.4.0p30
class      | fix
edition    | cre
component  | wato
level      | 1
compatible | yes

When editing a _Check TCP port connection_ rule with "Expected response time"
configured, the warning and critical thresholds were silently reset to 1/1000th
of their intended value after saving.

For example, setting a warning threshold of 1000 ms and saving would store the
value as 1.0 second, but on the next edit it would display as 1 ms instead of
1000 ms. Saving again would then store 1 ms permanently.

This also prevented rules created in Checkmk 2.3 from loading correctly in 2.4 —
the form would show a validation error instead of the previously configured values.

This has been fixed.
