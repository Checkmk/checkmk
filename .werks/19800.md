# Fix for writing disabled services into autochecks

During service discovery, services matched by a "Disabled services" rule were written to the autochecks file without the approval of a user.
only monitored services are supposed to be written to that file.

If the services on a host are short-lived and plentiful, e.g. like the containers in a K8S cluster, this led to an ever-growing autochecks file with thousands of entries and performance issues.

This werk prevents disabled services from being written to the autochecks file.
