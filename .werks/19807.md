# bi_aggregation: Fix state aggregation

key        | value
---------- | ---

class      | fix
edition    | community
component  | checks

Previously, the check plugin performed an additional state aggregation based on the collected WARN and CRIT states.
This circumvented the state restriction from the BI aggregation rule.
So, while the expected state was displayed in the BI tree, the service created by the check plugin could show a different one.

This change ensures that the service state is the state calculated by the BI aggregation.
