# Test connection to host: honor SNMPv3 credentials entered in the form

When running connection tests on the _Test connection to host_ page, any SNMPv3 credentials you entered in the form were silently ignored. The tests ran against the credentials already saved on the host, so editing the form had no effect. When the host had no stored SNMPv3 credentials, the tests failed with the misleading error `API Error: Decryption of SNMPv3 password failed.`

This has been fixed. The values you enter on the page are now forwarded correctly, and changes take effect on the next test run.
