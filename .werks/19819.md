# Remove ntop widgets from shared dashboards

version    | 3.0.0b1
class      | fix
edition    | pro
component  | ntopng_integration
level      | 1
compatible | yes

Previously, the ntop dashboard widgets ("ntop alerts", "ntop flows" and "ntop top talkers") were rendered on shared dashboards (those you create via "Share dashboard"), but they never actually showed any data there. The widgets could not authenticate against ntop on behalf of the user opening the share link, so they only displayed a login error in place of their content.

This has been fixed. On shared dashboards, ntop widgets now show a "Not available on shared dashboards" placeholder instead of the broken widget.

You don't need to do anything after the upgrade. Shared dashboards that already contain ntop widgets keep working, with the placeholder appearing where the broken widget used to be. On regular (non-shared) dashboards, ntop widgets continue to work as before.
