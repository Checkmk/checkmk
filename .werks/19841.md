# multipath: don't crash when numpaths is 0

key        | value
---------- | ---
date       | 2026-04-21T00:00:00+00:00
version    | 2.5.0p1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

When a multipath group had `numpaths` equal to `0` (for example because all
discovered paths had been removed from the device map between discovery and
the next check cycle), the check crashed with a `ZeroDivisionError` while
computing the active-path percentage.

The check now reports a CRIT state with "No paths" and continues to yield the
remaining information instead of crashing.
