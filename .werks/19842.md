# pfsense_if: skip interfaces with empty name during discovery

key        | value
---------- | ---
date       | 2026-04-21T00:00:00+00:00
version    | 2.5.0p1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

When the pfSense SNMP data included an interface entry with an empty name,
service discovery crashed with `TypeError: 'item' must be a non empty string
or ommited entirely, got ''` because `Service(item="")` is not allowed.

Empty-name entries are now skipped during parsing so that discovery yields
only valid services.
