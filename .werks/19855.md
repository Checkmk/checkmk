# Cisco Meraki: Validate empty "Sections" and "Cache per resource" options

In the agent configuration it was possible to select these options and not
select any items. Although this sort of configuration is unlikely, it is now
validated against so that you must select at least one if selected.

The main issue is with the "Sections" option and the behavior is likely not
what you expect. By not selecting any items, it actually defaults to the
following sections:

- API response codes
- Appliance uplink statuses
- Appliance VPN statuses
- Device statuses
- Device uplinks (HW/SW inventory only)
- Licenses overview
- Sensor readings
- Wireless ethernet statuses

In the rare event that you have selected one of these options and not selected
any items, please edit your configuration to either disable it or select at
least one section.
