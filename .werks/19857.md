# kube_pod_status: don't crash when an older status is still cached

key        | value
---------- | ---
date       | 2026-04-23T08:56:10.806054+00:00
version    | 2.5.0p1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

After upgrading a Kubernetes-monitored site, the pod *Status* service
could become `UNKNOWN` with a traceback ending in `KeyError: 'OOMKilled'`
(or another status name). This happened when the persistent check state
still remembered a pod status whose name is no longer in the set the
check currently tracks.

The check now tolerates an unknown cached status instead of crashing;
the affected pod simply continues with the currently observed status.
