# Kubernetes: Fix crash on Unknown pod condition status

key        | value
---------- | ---
date       | 2026-04-24T22:25:31.264097+00:00
version    | 2.6.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Previously, pods with a condition status of `Unknown` caused the Kubernetes
special agent to fail with a parsing error, breaking monitoring for the
entire cluster.

Such conditions are now handled correctly: they count against the time
threshold configured for that condition and alert when they persist.
