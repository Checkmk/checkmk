# Kubernetes Cluster collector service: Show and alert on cache metrics

---------- | ---
date       | 2026-05-01T02:15:03.101958+00:00
version    | 3.0.0b1
class      | feature
edition    | community
component  | checks
level      | 1
compatible | no

The Checkmk Kubernetes cluster collector now provides metrics about its
internal caches, which Checkmk monitors in the _Cluster collector_ service on
the Kubernetes source host.

As the cluster collector stores information about nodes and containers, the
cache size increases naturally over time. When the maximum cache size is
reached, gaps might appear in your graphs for Kubernetes related services.

The maximum cache size is configurable in the cluster collector Helm chart
(`clusterCollector.cacheMaxsize`).

You can now set alerts when the cache grows beyond a particular absolute size
or crosses a percentage threshold. As alerting is enabled by default, this Werk
is marked incompatible. The default thresholds for both cache types (container
metrics and machine sections) are to **warn at 80% full and go critical at
95%**. These can be configured or disabled in the _Kubernetes Collector info_
ruleset.
