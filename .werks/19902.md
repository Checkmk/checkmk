# Dashboard filter panel: adjust host and service filter categories

Five new groups have been introduced:

* **Cloud** — Azure subscription, subscription ID, resource group
* **Host properties** — host address (primary/IPv4/IPv6), address
  families, custom attributes, host group membership, service level,
  and similar host attributes
* **Agent** — time of registration, last contact and last download of
  the agent updater
* **Notifications** — notifications enabled and current notification
  number, for both hosts and services
* **Other** — number of services on a host, "has metrics" / "processes
  metrics", metric history, ntopng host

Additional previously-ungrouped filters were assigned to existing
categories: "Host state", "Service state", "Host in", "Service in",
"Service name", "Service check command", "Service is", and "Host check
command".

Some previously-grouped filters have been reorganized, and the
resulting empty groups have been removed:

* The five "Host address" filters now live in the new "Host properties"
  group. The "Host address" group has been removed.
* "Host is in Group" was moved into "Host properties" and "Host is
  stale" was moved into "Host state". The "Host is" group has been
  removed (its service-side counterpart "Service is" is unaffected).
* "Host name (regex)", "Host name (exact match)", and "Host name or
  alias (regex)" no longer sit inside a "Host name" group; they are
  rendered standalone at the top of the panel, alongside the
  already-ungrouped "Host alias (regex)", "Host tags", "Host labels",
  and "Host auxiliary tags". The "Host name" group has been removed.

The set of available filters and their behavior are unchanged. Search
across all filters works as before.
