# logwatch: Fix log lines reported multiple times after repeated service discovery

key        | value
---------- | ---
date       | 2026-05-06T13:55:57.765613+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Running service discovery multiple times on a new log file before the service
was created caused log lines to be re-parsed on every run. As a result, each
log line was reported as many times as service discovery had been executed.
Affected log lines now appear only once when the service is created.
