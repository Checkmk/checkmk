# Fix crash in check on unknown Palo Alto HA state 'initial'

key        | value
---------- | ---
date       | 2026-05-06T15:25:51.486032+00:00
version    | 2.5.0p2
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Previously, after an HA recovery, Palo Alto firewalls could briefly report
an `initial` HA peer or local state via SNMP. The _Palo Alto State_ check
plugin didn't recognize this state, causing a crash and generating a crash
report instead of handling it gracefully.

The plugin now maps the `initial` state to `WARN`, so the service
correctly shows `WARN` during this brief transition period.
