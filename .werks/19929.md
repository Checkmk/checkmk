# SAML connections: Record pending changes for REST API modifications

key        | value
---------- | ---
date       | 2026-06-18T09:29:08.910462+00:00
version    | 3.0.0b1
class      | fix
edition    | pro
component  | rest-api
level      | 1
compatible | yes

All SAML connection REST API endpoints that modify the configuration (create,
update and delete) now record a pending change and an audit log entry, as the
user interface already did. Previously, connections created or deleted via the
REST API were saved without showing up in "Activate pending changes", so they
were only synchronized to remote sites once an unrelated change was activated.
Remember to activate changes after modifying SAML connections via the REST API.
