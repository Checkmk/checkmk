# Cisco Meraki: fix appliance uplinks state map handling

class      | fix

The _Cisco Meraki appliance uplinks_ ruleset provides an option to override the
state of a given appliance uplink status, i.e. "active". Unfortunately, there
was a bug in how the "not connected" status was handled. The implementation is
now fixed. So, the override should now behave as it's supposed to.

No configuration is required. Simply upgrading to the target version should
resolve this issue.
