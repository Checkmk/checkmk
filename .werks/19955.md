# Fix Azure Storage latency graph displaying values in seconds instead of milliseconds

key        | value
---------- | ---
date       | 2026-05-11T19:15:01.033790+00:00
version    | 2.4.0p30
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

The graphs for **Azure/Storage performance** service metrics
`Success server latency` and `End-to-end latency` were displaying
values in seconds while the check output correctly showed them in
milliseconds.

The graphing unit has been corrected to milliseconds so the graph
scale now matches the check output.

This affects both the current Azure integration (`azure_v2`) and the
deprecated Azure integration.
