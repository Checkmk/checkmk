# Kaspersky AV Update Status: tolerate newer agent output

key        | value
---------- | ---
date       | 2026-05-22T10:05:29+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

Starting with Kaspersky Endpoint Security 12.3 for Linux, the agent reports
"Application databases loaded" instead of "Anti-virus databases loaded" and
no longer emits an "Anti-virus database records" line.  As a result, the
_AV Update Status_ service went UNKNOWN with `KeyError: 'Anti-virus databases
loaded'`.

The check now accepts either of the two key names for the loaded-status line
and skips the database-records line when it is not present.  The service
reports the available information again instead of crashing.
