# checkpoint_powersupply: Recognize "No Redundancy" as valid power supply status

After updating Check Point firewalls to R82, some power supplies report the status "No Redundancy".
Previously this status was not part of the default status mapping and was therefore treated as
`CRIT`.

"No Redundancy" is now recognized and maps to `WARN` by default, reflecting that the power supply
itself is healthy but redundancy is lost. You can configure the monitoring state for this status
via the rule _Check Point powersupply_. Devices that are intentionally connected to a single power
supply can map it to `OK` there.
