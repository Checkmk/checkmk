# checkpoint_tunnels: don't crash on unexpected tunnel status

version    | 2.5.0p7
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

The `checkpoint_tunnels` check monitors the state of Check Point VPN tunnels.

When a device reported a tunnel status that was not one of the known state
codes (for example a single field that combined several sub-tunnels), the
corresponding `Tunnel` service went to UNKNOWN with the message
"check failed - please submit a crash report!".

The check now tolerates such unexpected status values: the affected tunnel
service reports UNKNOWN together with the reported status text instead of
crashing, and the remaining tunnels continue to be monitored.
