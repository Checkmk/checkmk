# Redfish: allow connection timeout above 20 seconds

key        | value
---------- | ---
date       | 2026-06-19T09:56:38.735577+00:00
version    | 2.5.0p8
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

The timeout setting in the Redfish special agent rulesets _Redfish Compatible
Management Controller_ and _Redfish Compatible Power Equipment (PDU)_ was capped
at 20 seconds. Management controllers whose API responds more slowly than this
could not be monitored reliably: every request ran into the timeout, leaving
`BrokenPipeError` crash reports behind.

The maximum has been raised from 20 to 90 seconds, so you can now set a timeout
that matches slower hardware. Existing rules are unaffected.
