# Redfish: fix discovery crash for power supplies without a name

key        | value
---------- | ---
date       | 2026-06-30T07:07:42+00:00
version    | 2.4.0p34
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

When a management controller reported a power supply without a name, service
discovery of the *Power supply* services crashed. The crash aborted discovery
for the entire host, so the host produced a crash report and discovered no
services at all.

Such power supplies are now handled gracefully, and hosts that were affected by
the crash discover their power supply services again.

Power supplies that already have a name are unaffected and keep their existing
service, so no rediscovery is required for them.
