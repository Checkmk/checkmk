# Fix: Saving rule with predefined condition no longer drops the link

key        | value
---------- | ---
date       | 2026-05-20T09:36:55.597043+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | wato
level      | 1
compatible | no

Editing or creating a rule that referenced a predefined condition silently
turned the rule into one with explicit conditions. The link to the
predefined condition was lost on save, so later changes to the predefined
condition no longer affected those rules.

This affected newly created rules as well as pre-existing rules whenever
they were opened in the editor and saved again.

The fix only prevents the link from being dropped going forward. Rules that
already lost their link to a predefined condition before this update are not
restored automatically. To repair them, open each affected rule, select the
predefined condition again and save the rule.
