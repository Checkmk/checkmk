# Fix Check email migration of Cleanup messages option

Previously, _Check email_ rules that used "Forward mails as events to Event Console"
with "Cleanup messages" set to "Delete all processed messages" would fail to migrate
when updating. The update process would log errors like:

```
WARNING: Invalid rule configuration detected (Ruleset: active_checks:mail,
Title: Check email, ...)
WARNING: Config creation for active check mail failed on localhost: 3 validation
errors for Parameters
```

The affected rules would stop generating monitoring commands, causing the
corresponding services to go stale.

This has been fixed. Rules that were already partially migrated with the incorrect
internal value are also corrected automatically on the next config update.
