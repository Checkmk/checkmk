# Add automatic cleanup of the crash report directory

key        | value
---------- | ---
date       | 2026-06-23T06:02:54.226080+00:00
version    | 3.0.0b1
class      | feature
edition    | community
component  | multisite
level      | 2
compatible | yes

Checkmk now regularly cleans up the crash report directory
(`var/check_mk/crashes`). A daily background job removes crash reports older
than 90 days and, in addition to the existing per-crash-type count limit (200),
keeps the total size of all crash reports below 5 GB by deleting the oldest
reports first.

Previously crash reports were only pruned when a new crash of the same type
occurred, and only by count. A crash type that stopped producing new crashes was
therefore never cleaned up, and a single very large crash report was never
bounded by size. This could let the crashes directory grow to tens of gigabytes
and fill up the site's disk.

The cleanup runs automatically, no configuration is required.
