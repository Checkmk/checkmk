# Suppress site connection errors for sites not queried by a report

key        | value
---------- | ---
date       | 2026-05-25T14:08:50.320814+00:00
version    | 2.5.0p6
class      | fix
edition    | pro
component  | reporting
level      | 2
compatible | yes

When generating a report scoped to specific sites — either via a customer
context filter or explicit site filter on report elements — a disconnected
site outside that scope would previously cause a "Site Connection Errors"
page to appear, even though the report did not query that site at all. This
has been fixed.

The site connection error page has also been redesigned: when sites in scope
are unreachable, each affected site is now listed individually with its site
ID clearly highlighted, replacing the previous single combined WARNING block.
