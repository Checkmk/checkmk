# dell_idrac_disks, dell_idrac_virtdisks: do not crash on disks with empty name

key        | value
---------- | ---
date       | 2026-05-26T12:30:55.260685+00:00
version    | 2.4.0p32
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

The IDRAC MIB allows physical and virtual disks to report an empty
`physicalDiskName` / `virtualDiskName`. The discovery functions of
`dell_idrac_disks` and `dell_idrac_virtdisks` used this name as the service
item without checking, and crashed when the MIB returned an empty value
because `Service` rejects an empty item.

Discovery now skips disks with an empty name. Such disks remain unmonitored
until a follow-up change moves these plugins to index-based service items.
