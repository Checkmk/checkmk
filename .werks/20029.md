# dell_idrac_power: Support redefined value scope of powerUnitRedundancyStatus

With new iDRAC 10 a new SNMP MIB was introduced.
In this new MIB the values in powerUnitRedundancyStatus got refined.
This led to one of the following error cases:

* a crash, if the reported value was not part of the old MIB.
* a misleading service state, if the value means different things in the old and new MIB.

With this change, the plugin checks the iDRAC version and translates the redundancy status correctly.
For the new iDRAC 10 and older still supported ones.

To benefit from this change, no manual action is needed.
