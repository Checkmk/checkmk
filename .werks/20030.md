# automations: Write to logs

Previously, automations (e.g. autodiscovery) wrote messages to STDOUT, which got discarded later.
This can obfuscate any error message and leave issues unsolved.

With this change, the automations write to the logs instead.
Persisting (error) messages for analysis.
