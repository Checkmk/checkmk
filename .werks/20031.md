# dell_eql_storage: Stop crashing on (partially) empty rows

key        | value
---------- | ---
date       | 2026-06-02T07:41:57+00:00
version    | 2.4.0p32
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

The storage device is allowed to send no/empty data, while the plugin expected data.
This led to plugin crashes.

With this change, missing data is handled accordingly:

* Missing state information is displayed as "unknown".
* Missing values on storage usage lead to a stale service.
