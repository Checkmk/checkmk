# service discovery: Limit execution of active checks during/after discovery

key        | value
---------- | ---
date       | 2026-06-24T15:27:23+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

The execution of active checks is delayed until the result is required in the UI.
This led to duplicate items in a queue.
And the queue was processed multiple times while the UI polls for the service discovery result.
This caused a high load on a site.

This werk addresses the performance issues and ensures that active checks are executed only once after a service discovery is finished.
