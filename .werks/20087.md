# check-httpv2: reduce runtime overhead per check invocation

key        | value
---------- | ---
date       | 2026-06-02T13:41:26+00:00
version    | 3.0.0b1
class      | fix
edition    | cre
component  | checks
level      | 1
compatible | yes

Previously, each invocation of `check-httpv2` started a pool of worker
threads to handle the HTTP request.
Because the check only performs a single request, it could not benefit from the
additional threads.
On the contrary, starting the threads produced significant overhead related to
the number of CPUs of the system.

The check now runs on a single thread, reducing the CPU and memory overhead
of each invocation. This is particularly noticeable on sites that monitor
many HTTP endpoints.
