# Reverting changes now reverts OpenTelemetry collector config

key        | value
---------- | ---
date       | 2026-06-12T16:21:10.703790+00:00
version    | 2.5.0p8
class      | fix
edition    | ultimate

Previously, when you configured an OpenTelemetry collector under
"Setup > Telemetry" and then used _Revert changes_ before activating, the configuration was not removed.
Everything else was reverted as expected, but the OpenTelemetry collector
configuration files were left on disk, so the reverted configuration silently
survived.

This has been fixed.
