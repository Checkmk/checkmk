# Speed up Activate Changes in large distributed setups

key        | value
---------- | ---
date       | 2026-06-19T13:47:18.076411+00:00
version    | 2.4.0p33
class      | fix
edition    | cre
component  | wato
level      | 1
compatible | yes

When you activate changes in a distributed setup that are relevant to the piggyback-hub functionality, an updated piggyback configuration is sent to the remote sites.
Previously this was done one remote site after another, so activating changes became slow when many remote sites were connected.
This happened regardless of how many piggyback hosts you had, even with none configured at all.
Checkmk now sends this configuration to the remote sites in a single step.
As a result, activating changes is significantly faster in large distributed setups.
No configuration changes are required.
