# veritas_vcs: Do not report misleading state on cluster nodes without state data

key        | value
---------- | ---
date       | 2026-06-15T07:22:00.822389+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | checks
level      | 1
compatible | yes

On clustered hosts, when one node reported a Veritas service group with no
`State` entry (only `Frozen`/`TFrozen` rows, i.e. partial VCS output), the
_VCS Service Group_, _VCS System_ and _VCS Resource_ services on the cluster
showed a misleading `default` text for that node and were forced into the WARN
state.

A cluster node without state information now contributes no state to the
cluster verdict instead of a phantom `default`. The cluster state is then
derived solely from the nodes that did report a state and from any active
freezes.
