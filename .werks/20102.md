# Metric backend: replace three attribute lists with a single filter editor

When you select which OpenTelemetry data the metric backend should use, you no
longer fill in three separate lists for resource, scope and data point
attributes. Instead, a single filter editor lets you describe the telemetry you
are interested in by adding individual conditions.

You will find the new editor in the places where you select telemetry data:

* the custom graph editor, when adding a graph line that queries the metric
  backend,
* the Dynamic host management connector *OpenTelemetry data - Metric backend*,
  in the section *Select telemetry data for host creation*, and
* the special agent rule *Metric backend (custom query)*.

Each condition picks out telemetry by an attribute and its value, and Checkmk
selects the data that matches all of the conditions you add. The new editor
keeps your existing configuration intact, so there is nothing you need to adjust
after the upgrade.
