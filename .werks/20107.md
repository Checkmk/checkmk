# apc_symmetra: Fix time remaining shown 60 times too large in graph

version    | 2.5.0p8

component  | checks
level      | 1
compatible | yes

Since Checkmk 2.5.0 the graph of the _APC Symmetra status_ service showed the
"Time remaining" metric 60 times too large, while the value in the service
summary was correct.

The check plug-in records the metric in seconds since version 2.5.0 (it used
to be minutes up to 2.4), but the graph still applied the conversion from
minutes to seconds. The graph now renders the metric correctly.

Note for users upgrading from 2.4 or earlier: historic graph data of this
metric was recorded in minutes and will be displayed 60 times too small after
this fix. Graph data recorded with 2.5.0 or later is displayed correctly.
