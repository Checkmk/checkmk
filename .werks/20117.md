# Agent bakery: New "hotness" logic for "Bake agents" button

The agent bakery highlights the "Bake agents" and "Bake and sign agents" when new packages can be baked.
The logic to make the decision whether to highlight the button was based on a heuristic, and not always correct.
In particular:

* The button was wrongly not highlighted if chages to the password store made a rebake necessary.
* The button was wrongly highlighted if changes where made, but undone (for example deleting and restoring a rule).

The backend is now computing the actual agent hashes to ensure it correctly
highlights the button.
