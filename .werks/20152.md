# BI aggregations now support removing scheduled downtimes

version    | 3.0.0b1
class      | feature
edition    | community
component  | bi
level      | 1
compatible | yes

It was previously possible to schedule downtimes for all hosts and services
within a Business Intelligence aggregation via the _Commands_ dropdown in
aggregation views. However, there was no corresponding way to remove those
downtimes. Users had to navigate to individual host or service views to undo
the action.

The _Remove downtimes_ command is now available in the _Commands_ dropdown for
BI aggregation views. It removes all active downtimes for every host and
service that is a leaf of the selected aggregation.
